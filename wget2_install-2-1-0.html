Instructions for use of wget2 versions 2.1.0 and above with OpenSSL 3.0+

Use the OSS command
  tar xzf wget2-package.tar.gz -C / -p --no-same-owner
to install wget2. Make sure to have the directory containing the OpenSSL 3.0.x version of OpenSSL in your _RLD_LIB_PATH variable when wget2 is used. Typically, this is /usr/local-ssl3.0/lib.

You can build wget2 with other versions of OpenSSL. Make sure to change the various settings appropriately to reference the library area of OpenSSL.

Due to new features added in the latest OSS System Libraries, binaries built with those will may not function correctly on older RVUs. The following table lists the minimum RVU for each platform that can correctly run wget2 version 2.1.0, and above:

Platform Minimum RVU
TNS/E Not supported
TNS/X L23.08
TNS/V L23.08

Build Recipe

As of 2.1.0, wget2 requires the libpsl package. Please download and install it from ITUGLIB prior to attempting to build wget2.

To build wget2 from source, download the src package for wget2, and unzip it to a directory in OSS. The current build instructions used by ITUGLIB are:

tar xzf wget2-ssl3.0-2.1.0.tar.gz
CFLAGS="-c11" OPENSSL_CFLAGS="-I/usr/local-ssl3.0/include -L/usr/local-ssl3.0/lib" \
	OPENSSL_LIBS="-lssl -lcrypto" conf_script_floss_cc \
	--prefix=/usr/local-ssl3.0 --with-ssl=openssl
make
