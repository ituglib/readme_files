Instructions for use of Rsync versions 3.2.7 and above with SSL

Use the OSS command
  tar xzf rsync-package.tar.gz -C / -p --no-same-owner
to install rsync. Make sure to have the directory containing the OpenSSL 3.1.x, 3.0.x, 1.1.x, or 1.0.2u version of libcrypt.so and libssl.so in your _RLD_LIB_PATH or _RLD_FIRST_LIB_PATH variable when rsync is used.

If you are running on J-series, you must also include /usr/coreutils/lib in your RLD configuration.

Note: This package installs in a fixed path to ensure separation from incompatible SSL instances.

Due to new features added in the latest OSS System Libraries, binaries built with those will not function correctly on older RVUs. The following table lists the minimum RVU for each platform that can correctly run git version 3.2.7, and above:

Platform Minimum RVU
TNS/E J06.21
TNS/X L17.08
TNS/V L17.08

If you require new features or bug fixes that were released after 3.2.7, you will have to download the source package and build the new version yourself. This requires experience with FLOSS and with building open source tools on OSS.
