Instructions for use of nano versions 8.3 and above

Use the OSS command
  tar xzf nano-package.tar.gz -C / -p --no-same-owner
to install nano.

If you are using the wide (nanow) version, you must use the ncurses_32 package from ITUGLIB for the libncursesw.so DLL. For the standard version, only CoreUtils (T1202) is required. This may change over time.

Due to new features added in the latest OSS System Libraries, binaries built with those will not function correctly on older RVUs. The following table lists the minimum RVU for each platform that can correctly run nano version 8.3, and above:

Platform Minimum RVU
TNS/E J06.21
TNS/X L17.08
TNS/X - wide character L20.10
TNS/V L17.08
TNS/V - wide character L20.10

If you require new features or bug fixes that were released after 8.3, you will have to download the source package and build the new version yourself. This requires experience with FLOSS and with building open source tools on OSS.

If you don't need these new features you should continue to use version 8.3 instead of the latest version.
