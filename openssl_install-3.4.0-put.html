Instructions for use of OpenSSL 3.4.0 (PUT Library)

Use the OSS command
  tar xzf openssl-package.tar.gz -C / -p --no-same-owner
to install OpenSSL. You must install the package in the exact same location as in the tar file. Add the /usr/local-ssl3.4_put/lib64 for 64-bit or /usr/local-ssl3.4_put/lib for 32-bit to your _RLD_LIB_PATH variable when OpenSSL 3.4.0 PUT is needed. Other versions of OpenSSL should be installed elsewhere.

Note: Because of a problem with tar, the security of the /usr/local-ssl3.4_put and /usr/local-ssl3.4_put/share directories may be set to 0700 instead of 0755. To fix this, use the command
chmod 755 /usr/local-ssl3.4_put /usr/local-ssl3.4_put/share

Due to new features added in the latest OSS System Libraries, binaries built with those will not function correctly on older RVUs. The following table lists the minimum RVU for each platform that can correctly run OpenSSL version 3.4.0, and above:

Platform Minimum RVU
TNS/E J06.22
TNS/X L19.08
TNS/V L19.08

Caution: Do not mix versions of OpenSSL. There is no binary compatibility between version 1.1.1, 3.0.x, 3.1.x, 3.2.x, 3.3.x, and 3.4.x of OpenSSL. The first directory encountered in _RLD_LIB_PATH is selected by the OSS loader. Do not use a threaded library with an unthreaded program and vice versa.

Note: The TNS/X build may contain FIPS compatible providers. The ITUGLIB and OpenSSL teams do not formally perform FIPS certification on these builds. You must take responsibility for FIPS certification if it is required.
