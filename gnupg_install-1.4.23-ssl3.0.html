Instructions for use of GnuPG versions 1.4.23 and above with OpenSSL 3.0+

Use the OSS command
  tar xzf gnupg-package.tar.gz -C /usr/local-ssl3.0 -p --no-same-user
to install GnuPG. Make sure to have the directory containing the OpenSSL 3.0.x version of OpenSSL in your _RLD_LIB_PATH variable when GnuPG is used. Typically, this is /usr/local-ssl3.0.

Due to new features added in the latest OSS System Libraries, binaries built with those will may function correctly on older RVUs. The following table lists the minimum RVU for each platform that can correctly run GnuPG version 1.4.23, and above:

Platform Minimum RVU
TNS/E J06.22
TNS/X L20.10
TNS/V L20.10

Build Recipe

To build GnuPG from source, download the tarball for gnupg, and unzip it to a directory in OSS. The current build instructions used by ITUGLIB are:

tar xzf gnupg-1.4.23-src.tar.gz
cd gnupg-1.4.23
make
